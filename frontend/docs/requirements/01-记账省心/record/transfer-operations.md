## 功能概述

将记账页面顶部 tab 从「支出 | 收入」扩展为「支出 | 收入 | 转账」。

**核心原则：转账操作不是收支。** 转账、还债、借贷都是资金在账户间的转移（底层统一为账户间转账），不应计入收支统计。此前转账和还债藏在支出分类里，既不直观又容易混淆统计。

### 行业依据
- **鲨鱼记账**：独立债务追踪模块，支持借出/借入、还款进度、逾期提醒
- MoneyWiz/MoneyThings/YNAB 等专业记账APP均将"非收支操作"独立为第三类
- 钱迹明确声明："转账、还款、借入、借出都只是单纯的资金转移，并没有产生具体的收支行为"
- CountAbout 是行业唯一实现全自动本金/利息拆分的APP

### 用户故事

| 序号 | 用户角色 | 场景 | 需求 | 价值 |
|------|---------|------|------|------|
| 1 | 普通用户 | 从储蓄卡转钱到微信 | 快速完成账户间资金转移 | 便捷 |
| 2 | 信用卡用户 | 每月还信用卡账单 | 一键还款，不影响收支统计 | 准确 |
| 3 | 房贷用户 | 还房贷时自动拆分本金利息 | 利息自动计入支出，本金冲减负债 | 专业 |
| 4 | 经常借钱给朋友的用户 | 记录借出和收回 | 自动追踪债权，不影响收支 | 清晰 |
| 5 | 有私人借款的用户 | 记录借入和偿还 | 自动追踪债务，不影响收支 | 清晰 |

## 各操作类型详细设计

### 1. 转账

**场景**：储蓄卡→微信、现金存银行、支付宝→银行卡

**账户变化**：
```
转出账户余额：↓
转入账户余额：↑
总资产：不变
总负债：不变
净值：不变
收支统计：不影响
```

**交互**：
```
┌─────────────────────────────────┐
│  🔄 转账                        │
│                                  │
│  从 [  储蓄卡  ▼ ]             │
│  到 [  微信零钱  ▼ ]           │
│                                  │
│         ¥ 0.00                   │
│                                  │
│  备注：点击填写                  │
│                                  │
│  [        确认转账       ]       │
└─────────────────────────────────┘
```

**账户选择规则**：
- 转出账户：全部账户可选
- 转入账户：全部账户可选，排除已选转出账户

**后端处理**：
- 生成1条 `type=transfer` 记录
- 转出账户余额减少，转入账户余额增加

---

### 2. 还信用卡

**场景**：用储蓄卡还信用卡账单

**账户变化**：
```
还款账户（资产）余额：↓
信用卡账户（负债）余额：↑（负得少了，欠款减少）
总资产：↓（还款账户减少）
总负债：↓（信用卡欠款减少）
净值：不变（资产减少 = 负债减少）
收支统计：不影响（刷卡消费时已记过支出）
```

**交互**：
```
┌─────────────────────────────────┐
│  💳 还信用卡                    │
│                                  │
│  从 [  储蓄卡  ▼ ]             │  ← 仅资产类账户
│  还 [  信用卡  ▼ ]             │  ← 仅信用卡类账户
│                                  │
│         ¥ 0.00                   │
│                                  │
│  备注：点击填写                  │
│                                  │
│  [        确认还款       ]       │
└─────────────────────────────────┘
```

**账户选择规则**：
- 还款账户：仅资产类（cash/investment/fixed_asset/depreciable_asset）
- 信用卡账户：仅信用卡类（credit_card）

**后端处理**：
- 生成1条 `type=repayment` 记录
- 还款账户余额减少，信用卡余额增加（负值变少）

**与原"还债"的区别**：
- 原来的"还债"统一处理所有负债，现在拆分为"还信用卡"和"还贷款"
- 信用卡还款不需要拆分本金/利息（刷卡消费时已记支出）
- 信用卡还款更简单：全额还款即可

---

### 3. 还贷款（⭐ 核心功能：本金+利息拆分）

**场景**：还房贷、车贷、消费贷等

**账户变化**：
```
还款账户（资产）余额：↓（扣除还款总额）
贷款账户（负债）余额：↓（仅本金部分减少）
利息支出：↑（自动生成一笔支出记录）
总资产：↓（还款账户减少）
总负债：↓（本金部分减少）
净值：↓（利息部分是真实支出）
收支统计：仅利息部分计入支出
```

**交互**：
```
┌─────────────────────────────────┐
│  🏠 还贷款                      │
│                                  │
│  从 [  储蓄卡  ▼ ]             │  ← 仅资产类账户
│  还 [  房贷-建行  ▼ ]          │  ← 仅负债类账户（liability）
│                                  │
│         ¥ 5,000.00              │
│                                  │
│  ── 本金利息拆分 ──             │
│                                  │
│  ● 自动拆分（推荐）             │
│  ┌─────────────────────────┐    │
│  │ 本金：¥3,200  (自动计算) │    │
│  │ 利息：¥1,800  (自动计算) │    │
│  └─────────────────────────┘    │
│                                  │
│  ○ 手动拆分                     │
│                                  │
│  利息分类：[住房利息      ▼]    │  ← 利息记入哪个支出分类
│  备注：点击填写                  │
│                                  │
│  [        确认还款       ]       │
└─────────────────────────────────┘
```

**拆分逻辑**：

| 条件 | 拆分方式 |
|------|---------|
| 贷款账户已配置参数（本金/利率/期数/方式） | 自动拆分：根据摊销表计算当期本金和利息 |
| 贷款账户未配置参数 | 默认手动拆分，用户自行填写本金和利息金额 |
| 金额 = 月供 | 按摊销表精确拆分 |
| 金额 > 月供 | 多还部分全部冲减本金 |
| 金额 < 月供 | 优先还利息，剩余还本金 |

**自动拆分计算公式**（复用 LiabilityForm 已有逻辑）：

```
等额本息：
  月供 M = P × [i(1+i)^n] / [(1+i)^n - 1]
  当期利息 = 剩余本金 × 月利率
  当期本金 = M - 当期利息

等额本金：
  当期本金 = 原始本金 / 总期数
  当期利息 = 剩余本金 × 月利率

先息后本：
  非最后一期：本金 = 0，利息 = 剩余本金 × 月利率
  最后一期：本金 = 剩余本金，利息 = 剩余本金 × 月利率
```

**后端处理**：
- 生成2条记录：
  - ① `type=repayment`，金额=本金部分 → 还款账户余额↓，贷款账户余额↓（本金减少）
  - ② `type=expense`，金额=利息部分 → 还款账户余额↓（再次减少），分类=用户选择的利息分类
- 两笔共享同一日期和备注（备注可追加"利息部分"标识）

### 4. 借出/收回

**场景**：借钱给朋友、垫付团建费用、收回借款

**借出 - 账户变化**：
```
现金账户余额：↓
应收账款（虚拟账户）余额：↑
总资产：不变（现金→债权，资产形态变化）
净值：不变
收支统计：不影响
```

**收回借款 - 账户变化**：
```
现金账户余额：↑
应收账款（虚拟账户）余额：↓
总资产：不变（债权→现金，资产形态变化）
净值：不变
收支统计：不影响
```

**交互**：
```
┌─────────────────────────────────┐
│  🤝 借出/收回                   │
│                                  │
│  ┌──────────┐ ┌──────────┐     │
│  │  借出    │ │  收回    │     │  ← 方向切换
│  └──────────┘ └──────────┘     │
│                                  │
│  对方：[  张三  ▼/输入 ]        │  ← 选择或新建联系人
│  从 [  储蓄卡  ▼ ]             │  ← 借出：资产类账户
│                                  │
│         ¥ 0.00                   │
│                                  │
│  备注：点击填写                  │
│                                  │
│  [        确认借出       ]       │
└─────────────────────────────────┘
```

**应收账款（隐式账户）**：
- 用户选择"借出给张三"时，系统自动创建/更新 `应收-张三` 虚拟账户
- 类型为 `receivable`（资产类），用户无需手动管理
- 在资产总览页的"债权"分组下展示
- 收回时自动抵减对应应收账款余额

**后端处理**：
- 借出：生成1条 `type=transfer` 记录，from=现金账户，to=应收账款
- 收回：生成1条 `type=transfer` 记录，from=应收账款，to=现金账户

---

### 5. 借入/偿还

**场景**：找朋友借钱、网贷到账、偿还私人借款

**借入 - 账户变化**：
```
现金账户余额：↑
应付账款（虚拟账户）余额：↑（债务增加）
总资产：↑
总负债：↑
净值：不变
收支统计：不影响
```

**偿还借款 - 账户变化**：
```
现金账户余额：↓
应付账款（虚拟账户）余额：↓（债务减少）
总资产：↓
总负债：↓
净值：不变
收支统计：不影响
```

**交互**：
```
┌─────────────────────────────────┐
│  📋 借入/偿还                   │
│                                  │
│  ┌──────────┐ ┌──────────┐     │
│  │  借入    │ │  偿还    │     │  ← 方向切换
│  └──────────┘ └──────────┘     │
│                                  │
│  对方：[  李四  ▼/输入 ]        │  ← 选择或新建联系人
│  到 [  储蓄卡  ▼ ]             │  ← 借入：资产类账户
│                                  │
│         ¥ 0.00                   │
│                                  │
│  备注：点击填写                  │
│                                  │
│  [        确认借入       ]       │
└─────────────────────────────────┘
```

**应付账款（隐式账户）**：
- 用户选择"从李四借入"时，系统自动创建/更新 `应付-李四` 虚拟账户
- 类型为 `payable`（负债类），用户无需手动管理
- 在资产总览页的"债务"分组下展示
- 偿还时自动抵减对应应付账款余额

**后端处理**：
- 借入：生成1条 `type=transfer` 记录，from=应付账款，to=现金账户
- 偿还：生成1条 `type=transfer` 记录，from=现金账户，to=应付账款

参考鲨鱼记账的债务追踪模块设计：


### 还贷款拆分请求

```typescript
interface RepaymentSplitRequest {
  accountId: number           // 还款账户（资产类）
  toAccountId: number         // 贷款账户（负债类）
  totalAmount: number         // 还款总额
  principalAmount: number     // 本金部分
  interestAmount: number      // 利息部分
  interestTypeId?: number     // 利息支出的分类ID
  splitMode: 'auto' | 'manual'  // 拆分方式
  date: string
  remark?: string
}
```

### 隐式账户结构

```typescript
interface ImplicitAccount {
  id: string                  // 账户ID
  name: string                // 格式：应收-张三 / 应付-李四
  type: 'receivable' | 'payable'
  counterparty: string        // 对方名称
  balance: number             // 当前余额
  isImplicit: true            // 标记为隐式账户
  isVisible: false            // 不在账户列表页显示（仅在资产总览债权/债务分组展示）
  createdAt: Date             // 创建时间
  updatedAt: Date             // 更新时间
}
```

## 明细页展示规则

转账/还债/借贷记录在明细页（BillCard）中展示，与账户金额调整记录同一层级：

- **显示位置**：按日期分组，在明细页记录列表中正常展示
- **显示前缀**：使用 `→` 符号，区别于收入的 `+` 和支出的 `-`
- **金额样式**：中性灰色（`--color-text-secondary`），与调整记录一致
- **默认名称**：
  - 转账 → "转账"
  - 还信用卡 → 使用备注，若无则显示 "还款"
  - 还贷款 → 使用备注，若无则显示 "还款"
  - 借出/借入 → 使用备注，若无则显示转账类型名称
- **收支统计**：不计入日/月/年收支汇总，不影响收入、支出、结余计算
- **分类修改**：不支持修改分类（转账记录无 typeId）

## 边界情况

1. **同账户转账**：转出和转入相同，提示禁止
2. **还贷款金额 < 利息**：提示"还款金额不足以覆盖当期利息"
3. **还贷款金额 > 月供**：多还部分全部冲减本金，提示"本期多还本金 ¥xxx，可缩短x期"
4. **无信用卡账户时选还信用卡**：提示"请先创建信用卡账户"
5. **无负债账户时选还贷款**：提示"请先创建贷款账户"
6. **借出/借入对方名重复**：同名视为同一人，复用已有隐式账户
7. **收回金额 > 应收余额**：提示"收回金额超过应收余额 ¥xxx"
8. **偿还金额 > 应付余额**：提示"偿还金额超过应付余额 ¥xxx"
9. **隐式账户删除**：应收/应付账户余额为0时可删除，有余额不可删除
10. **贴息处理**：还贷款时贴息金额计入本金部分（已由 LiabilityForm 支持）
11. **弹框高度溢出**：还贷款表单内容较多时，弹框高度限制在 80-85vh，内容区域可滚动，遮罩层可正常点击关闭

## UI/UX 优化

### 弹框高度与滚动优化

**问题**：还贷款页面弹框高度可能超过一屏，导致无法点击遮罩层退出表单编辑页面

**解决方案**：
- **WdPopup 配置**：
  - 添加 `:safe-area-inset-bottom="true"` 适配安全区域
  - 弹框最大高度限制为 `85vh`，防止内容超出屏幕
  - 弹框内容区域 `overflow: hidden`，避免双重滚动
- **TransactionForm 组件**：
  - 内容区域最大高度为 `80vh`
  - 启用垂直滚动 `overflow-y: auto`
  - 添加 `-webkit-overflow-scrolling: touch` 优化移动端滚动体验

**验收标准**：
- [x] 还贷款表单内容超过屏幕高度时可正常滚动
- [x] 弹框顶部和底部在各种屏幕尺寸下保持可见
- [x] 点击弹框外部遮罩层可正常关闭弹框
- [x] 移动端安全区域适配正确，底部内容不被遮挡
